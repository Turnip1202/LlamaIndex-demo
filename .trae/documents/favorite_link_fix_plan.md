# 收藏链接原始分类ID存储方案

## 问题分析

**当前问题**：在收藏分类中取消收藏时，系统需要动态查找链接的原始分类，可能导致错误地操作其他链接。

**根本原因**：收藏分类中的链接缺少原始分类ID信息，每次操作都需要遍历查找。

**用户建议**：为收藏的链接添加专门的属性来存储原始分类ID，确保在收藏分类中操作时能直接使用正确的分类ID。

## 实现计划

### 1. 数据结构修改

**修改文件**：`/workspace/src/types/index.ts`

**修改内容**：
- 为 `Link` 接口添加 `originalCategoryId` 属性，用于存储链接的原始分类ID
- 确保该属性为可选，以保持向后兼容性

### 2. LinksManager 修改

**修改文件**：`/workspace/src/utils/links/LinksManager.ts`

**修改内容**：
- 在 `addLink` 方法中，为新添加的链接设置 `originalCategoryId` 属性
- 在 `toggleFavorite` 方法中，确保 `originalCategoryId` 属性被正确保存
- 在 `getFavoriteLinks` 方法中，返回包含原始分类ID的收藏链接信息

### 3. LinkGrid 组件修改

**修改文件**：`/workspace/src/components/base/LinkGrid.tsx`

**修改内容**：
- 在收集收藏链接时，确保每个链接都有 `originalCategoryId` 属性
- 在收藏分类中渲染链接时，直接使用链接的 `originalCategoryId` 属性
- 简化处理逻辑，不再需要动态查找原始分类

### 4. 数据迁移

**修改内容**：
- 添加数据迁移逻辑，为现有的收藏链接补全 `originalCategoryId` 属性
- 确保在应用启动时自动执行迁移

## 预期效果

1. **收藏操作**：当用户收藏链接时，系统自动记录链接的原始分类ID
2. **取消收藏操作**：在收藏分类中取消收藏时，直接使用链接的 `originalCategoryId` 属性
3. **性能优化**：避免了每次操作都需要遍历查找原始分类的开销
4. **可靠性提升**：确保操作始终使用正确的分类ID，避免误操作

## 风险评估

**潜在风险**：
- 数据迁移可能会影响现有数据
- 修改数据结构可能需要更新其他使用链接数据的组件

**风险缓解**：
- 数据迁移时添加备份机制
- 保持向后兼容性，确保旧数据仍然可以正常工作
- 充分测试确保修改不会影响其他功能

3. 修改 `LinkGrid.tsx` 使用新属性简化逻辑
4. 添加数据迁移逻辑
5. 测试收藏和取消收藏功能
6. 构建并推送修改
