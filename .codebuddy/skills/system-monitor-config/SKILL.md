# 系统监控配置技能

## 功能概述

系统监控配置是 RPA 系统中用于配置各类异常监控规则的核心模块，主要功能包括：

- **监控规则配置**：配置各类异常的监控规则，决定什么情况下自动生成工单
- **子表管理**：管理不同类型监控的子表数据，如城市配置、网站配置等
- **导入导出**：支持配置的导入导出，方便批量操作
- **综合查询**：查询系统配置和客户配置的完整视图

## 核心概念

### 监控类型

| 监控类型编码 | 监控类型名称 | 说明 |
|---|---|---|
| 40010001 | 网站异常 | 定时 ping 当前流程使用的网站，ping 不通即触发 |
| 40010002 | 申报数据异常 | 数据在【待申报】或【申报中】状态停留时间超过城市配置阈值 |
| 40010003 | 设备离线 | 设备状态匹配 trigger_rule 中配置的状态 |
| 40010004 | 流程重复执行 | 流程失败次数达到配置阈值 |
| 40010005 | 缴费异常 | 缴费任务超过配置时间 |
| 40010006 | Ukey未插入 | 已绑定 Ukey 状态匹配 trigger_rule |

### 数据结构

系统监控配置采用主表+子表的设计模式：

- **主表**：`ticket_monitor_config`，存储监控配置基本信息
- **子表**：按监控类型分表，存储具体监控规则

## 核心业务逻辑

### 配置管理

#### 新增/更新配置
- 新增：插入主表 → 保存子表数据
- 更新：更新主表 → 删除子表数据 → 保存子表数据（先删后插）

#### 删除配置
- 先查询主表获取 `ticketType`
- 根据 `ticketType` 删除对应子表数据
- 删除主表数据

### 子表更新规则
- **有独立 CRUD 接口的子表**（申报数据异常城市配置、Ukey城市监控配置）：
  - `updateXxx` 不操作这两类子表
  - `deleteXxx` 时单独级联删除
- **无独立接口的子表**（通知人、网站配置、流程重复、缴费异常）：
  - 统一先删后插（`deleteSubTableData` + `saveSubTableData`）

### 导入导出

#### 申报数据异常城市配置导入
- 解析 Excel 文件
- 业务类型转换："社保"→"1"、"公积金"→"2"
- 城市名查询 `addrCode`
- `(addrName + "_" + businessType)` 去重，部分成功模式

#### Ukey城市监控配置导入
- 解析 Excel 文件
- 按城市名分组聚合：
  - 出现"社保"行 → `socialMonitor=1`
  - 出现"公积金"行 → `fundMonitor=1`
- 同城市只插一条记录，去重 key 为 `addrCode`

### 综合查询

#### `getFullConfig` 接口
- 按 `ticketType + status=1` 查询系统监控配置
- 填充系统配置对应子表
- 按 `customerId + ticketType` 查询客户配置
- 填充客户配置对应子表
- 返回完整视图 `TicketMonitorFullConfigVO`
