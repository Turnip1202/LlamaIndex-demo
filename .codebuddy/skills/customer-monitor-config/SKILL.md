# 客户监控配置技能

## 功能概述

客户监控配置是 RPA 系统中用于客户自定义监控规则的模块，主要功能包括：

- **客户规则配置**：客户对监控规则的自定义覆盖，与系统监控配置逻辑独立
- **子表管理**：管理客户配置的子表数据，如城市明细、流程重复执行次数等
- **导入导出**：支持客户配置的导入导出，方便批量操作
- **综合查询**：与系统配置一起查询，形成完整视图

## 核心概念

### 客户配置类型

| 配置类型编码 | 配置类型名称 | 说明 |
|---|---|---|
| 40010002 | 申报数据异常 | 客户自定义城市+业务类型的时间阈值 |
| 40010004 | 流程重复执行 | 客户自定义各服务项目的失败次数阈值 |
| 40010005 | 缴费异常 | 客户自定义各缴费状态的时间阈值 |
| 40010006 | Ukey未插入 | 客户自定义城市粒度的社保/公积金监控开关 |

### 数据结构

客户监控配置采用主表+子表的设计模式：

- **主表**：`ticket_monitor_customer_config`，存储客户配置基本信息
- **子表**：按配置类型分表，存储具体自定义规则

## 核心业务逻辑

### 配置管理

#### 新增/更新配置
- 新增：插入主表 → 保存子表数据
- 更新：更新主表 → 删除子表数据 → 保存子表数据（先删后插）

#### 删除配置
- 删除主表数据
- 级联删除对应子表数据

### 子表更新规则
- **有独立 CRUD 接口的子表**（申报数据异常城市明细、Ukey城市监控配置）：
  - `updateXxx` 不操作这两类子表
  - `deleteXxx` 时单独级联删除
- **无独立接口的子表**（流程重复执行、缴费异常）：
  - 统一先删后插（`deleteCustomerSubTableData` + `saveCustomerSubTableData`）

### 导入导出

#### 申报数据异常城市明细导入
- 解析 Excel 文件
- 业务类型转换："社保"→"1"、"公积金"→"2"
- 城市名查询 `addrCode`
- `(addrName + "_" + businessType)` 去重，部分成功模式

#### Ukey城市监控配置导入
- 解析 Excel 文件
- 按城市名分组聚合：
  - 出现"社保"行 → `socialMonitor=1`
  - 出现"公积金"行 → `fundMonitor=1`
- 同城市只插一条记录，去重 key 为 `addrCode`

### 综合查询

#### 与系统配置关联
- 触发工单时，先按 `customer_id + ticket_type` 查客户配置
- 有则用客户自定义规则，无则降级使用全局监控配置规则
- `getFullConfig` 接口返回系统配置和客户配置的完整视图
