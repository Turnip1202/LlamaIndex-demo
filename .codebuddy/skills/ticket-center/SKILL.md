# 工单中心技能

## 功能概述

工单中心是 RPA 系统中用于管理各类异常的核心模块，主要功能包括：

- **工单生成**：基于监控配置自动生成各类异常工单
- **工单管理**：处理、查询、批量操作已生成的工单
- **工单查询**：支持按类型、状态、时间等条件查询工单
- **明细查询**：获取工单对应的详细信息（如网站异常、设备离线等）

## 核心概念

### 工单类型

| 工单类型编码 | 工单类型名称 | 说明 |
|---|---|---|
| 40010001 | 网站异常 | 网站访问异常，定时 ping 检测 |
| 40010002 | 申报数据异常 | 申报数据在待申报/申报中状态超时 |
| 40010003 | 设备离线 | 设备状态异常，如离线、关闭等 |
| 40010004 | 流程重复执行 | 流程失败次数达到阈值 |
| 40010005 | 缴费异常 | 缴费任务超过配置时间 |
| 40010006 | Ukey未插入 | Ukey 状态异常 |

### 数据结构

工单中心采用主表+子表的设计模式：

- **主表**：`ticket_center`，存储工单基本信息
- **子表**：按异常类型分表，存储具体异常详情

## 核心业务逻辑

### 工单生成逻辑

#### 申报数据异常 (`batchInsertTicket`)
- 去重 key：`ticket_type + declare_account + customer_id + business_type + addr_code + declare_type`
- 未处理工单：只更新 `reportTime` 及明细计数
- 已处理工单：重新插入
- 不存在：全新插入
- 处理人：从 `dev_user_addr` 按城市+业务类型查询

#### 其他类型 (`batchInsertTicketOther`)
- 支持：网站异常、设备离线、流程重复执行、缴费异常、Ukey未插入
- 去重 key：`ticket_type + customer_id + business_type + addr_code`
- 未处理工单：只更新 `reportTime`
- 已处理工单：重新插入
- 不存在：全新插入

### 处理人/通知人来源
- 网站异常、流程重复执行、缴费异常：从 `dev_user_addr` 按城市+业务类型查询
- 设备离线、Ukey未插入：从 `ticket_monitor_notify_person` 表查询（多人）

### 工单编号
- 格式：`yyyyMMdd + 8位当日自增序号`
- 使用 Redis `INCR` 保证唯一，key 次日 0 点过期

### 优先级升级
- 定时任务每 30 分钟执行
- 每超 2 小时升一级（3→2→1）
- 升级后通知处理人

## 查询功能

### `getTicketById` 接口
- 根据工单 ID 查询工单详情
- 同时查询对应子表数据
- 支持所有类型工单的明细查询

### 列表查询
- 支持分页查询
- 支持按工单类型、状态、时间等条件过滤
